# Findings: Duplicate Wallet Import Issue

**Date:** January 11, 2026  
**Component:** AddCurrencyModal - Wallet Import Functionality  
**Status:** ✅ Fixed

---

## Problem Summary

When importing wallet addresses from JSON files using the AddCurrencyModal component, duplicate wallets are being created in the WalletsTab view, even when wallets with the same address and currency already exist for a contact.

---

## Error Location

**File:** `src/components/modals/AddCurrencyModal.vue`  
**Function:** `handleFileImport()`  
**Related Files:**
- `src/services/walletService.ts` - Wallet creation service
- `src/components/tabs/contactDetails/WalletsTab.vue` - Wallet display component

---

## Root Cause Analysis

### Current Import Flow

1. User clicks "Import" button in AddCurrencyModal
2. File picker opens for JSON file selection
3. JSON file is parsed to extract wallet data
4. User confirms import in WalletImportConfirmModal
5. `handleConfirmImport()` is called:
   - Fetches existing wallets and checks for duplicates ✅
   - Adds only new wallets via `addWallet()` ✅
   - **BUT THEN** emits `currency-added` event for each wallet ❌
6. `handleCurrencyAdded()` in ContactDetailsModal.vue receives the event:
   - Calls `addWallet()` AGAIN without duplicate checking ❌
   - This creates duplicates!

### The Problem

**Double-Adding Issue:** Wallets are being added TWICE:

1. **First Addition (Correct):** In `handleConfirmImport()`, wallets are added with proper duplicate checking
2. **Second Addition (Incorrect):** The `currency-added` event is emitted for each wallet, which triggers `handleCurrencyAdded()` in ContactDetailsModal.vue, which adds the wallets AGAIN without duplicate checking

**Root Cause:**
- `handleConfirmImport()` correctly filters duplicates before adding wallets
- However, it then emits `currency-added` events for each new wallet
- `handleCurrencyAdded()` in ContactDetailsModal.vue receives these events and calls `addWallet()` again without checking for duplicates
- This results in every imported wallet being added twice (once correctly, once as a duplicate)

### Code Flow

**AddCurrencyModal.vue - handleConfirmImport():**
```typescript
// ✅ Correctly checks for duplicates
const existingWallets = await getWalletsForContact(props.contactId);
const newWallets = wallets.filter(wallet => !walletExists(...));

// ✅ Correctly adds only new wallets
for (const wallet of newWallets) {
  await addWallet({ ... });  // First addition - CORRECT
  
  // ❌ PROBLEM: Emits event that triggers second addition
  emits('currency-added', { ... });
}
```

**ContactDetailsModal.vue - handleCurrencyAdded():**
```typescript
// ❌ PROBLEM: Adds wallet again without duplicate checking
const handleCurrencyAdded = async (currencyData) => {
  await addWallet({  // Second addition - CREATES DUPLICATE
    contactId: currencyData.contactId,
    coinTicker: currencyData.network,
    address: currencyData.address
  });
};
```

This results in:
- Multiple wallets with identical addresses being created
- Duplicate entries appearing in WalletsTab.vue
- Unnecessary database entries
- Potential data inconsistency

### Code Flow

**AddCurrencyModal.vue - handleFileImport():**
```typescript
// JSON is parsed
const importedWallets = jsonData.addresses.map(addr => ({
  coinTicker: addr.currency,
  address: addr.address,
  keyFingerprint: addr.keyFingerprint,
  cryptoPublicKey: addr.cryptoPublicKey,
  gpgPublicKey: addr.gpgPublicKey,
  // ... other fields
}));

// Each wallet is added without checking for duplicates
for (const walletData of importedWallets) {
  await addWallet(props.contactId, walletData);
}
```

**walletService.ts - addWallet():**
- The `addWallet()` function creates a new wallet entry directly
- No validation to check if a wallet with the same address and coinTicker already exists
- Each call results in a new database entry

## Expected Behavior

The import functionality should:
1. Check if a wallet with the same `address` and `coinTicker` already exists for the contact
2. Skip creation if a duplicate is found
3. Only create wallets that don't already exist
4. Optionally provide feedback to the user about:
   - How many wallets were imported
   - How many were skipped (duplicates)
   - Any errors encountered

## Recommended Solution

### Option 1: Remove currency-added Event Emission (Recommended)
**Root Cause Fix:** Stop emitting `currency-added` events from `handleConfirmImport()` since wallets are already being added directly:

1. Remove the `emits('currency-added', ...)` calls from `handleConfirmImport()`
2. The wallets are already added correctly with duplicate checking
3. The `currency-added` event should only be used for single wallet additions via the form
4. Manually refresh the wallet count/list after import completes

**Rationale:**
- Fixes the root cause (double-adding)
- Simplifies the code flow
- Maintains correct behavior for single wallet additions
- No need to modify ContactDetailsModal

### Option 2: Add Duplicate Checking to handleCurrencyAdded
Add duplicate checking to `handleCurrencyAdded()` in ContactDetailsModal.vue:

1. Fetch existing wallets before adding
2. Check if wallet already exists
3. Only add if it doesn't exist
4. This prevents duplicates at the event handler level

**Rationale:**
- Provides defense-in-depth
- Protects against duplicates from any source
- More robust but adds complexity

### Option 3: Add Validation to walletService
Add duplicate checking logic to `addWallet()` function:
- Check for existing wallet before creation
- Return error or skip if duplicate found
- This would prevent duplicates at the service level

**Rationale:**
- Prevents duplicates at the lowest level
- Protects all code paths
- Most robust but changes core service API

## Solution Implemented

### Fix Applied

**Root Cause:** Double-adding wallets due to event emission triggering duplicate addition

**Solution:**
1. ✅ Removed `currency-added` event emission from `handleConfirmImport()` 
   - Wallets are already added directly via `addWallet()`
   - Emitting events caused `handleCurrencyAdded()` to add them again
2. ✅ Added new `wallets-imported` event for bulk import completion
   - Notifies parent to refresh wallet list without re-adding wallets
3. ✅ Added duplicate checking to `handleCurrencyAdded()` for single wallet additions
   - Prevents duplicates when adding wallets via the form
   - Provides user feedback if duplicate is detected

### Code Changes

**AddCurrencyModal.vue:**
- Removed `emits('currency-added', ...)` from `handleConfirmImport()`
- Added `wallets-imported` event emission after successful import
- Added comment explaining why `currency-added` is not emitted for bulk imports

**ContactDetailsModal.vue:**
- Added `@wallets-imported` event handler
- Added `handleWalletsImported()` function to refresh wallet list
- Added duplicate checking to `handleCurrencyAdded()` for single wallet additions

---

## Conclusion

The duplicate wallet import issue was caused by **double-adding wallets**:
1. First addition: `handleConfirmImport()` correctly added wallets with duplicate checking
2. Second addition: `currency-added` events triggered `handleCurrencyAdded()` which added wallets again without checking

**Fix:** Removed unnecessary event emission from bulk import and added proper refresh mechanism. Also added duplicate checking to single wallet addition handler for defense-in-depth.
