## 6. Migration Notes for Future Development

### 6.1. Import Path Patterns

**Always use these patterns:**

```typescript
// Export functionality
import { ... } from "@/lib/exportStandard/currencies";

// Import functionality
import { ... } from "@/lib/importStandard/fileImports.seed";

// Individual currency GPG generation
import { generateDeterministicGPGKey } from "@/lib/cryptographyCore/deterministicGPG/deterministicGPG.individual";

// Seed-level GPG generation
import { generateGPGFromRootExtendedPrivateKey } from "@/lib/cryptographyCore/deterministicGPG/deterministicGPG.seed";

// Utility functions (still in utils)
import { generateAddressesFromMnemonic } from "@/utils/cryptoGenerator";
```

### 6.2. Adding New GPG Generation Methods

If you need to add a new GPG generation method:

1. Create new file: `src/lib/cryptographyCore/deterministicGPG/deterministicGPG.{type}.ts`
2. Follow the existing patterns from `.individual.ts` or `.seed.ts`
3. Export functions with clear names
4. Import using: `@/lib/cryptographyCore/deterministicGPG/deterministicGPG.{type}`

### 6.3. Adding New Export Formats

If you need to add new export formats or types:

1. Add to: `src/lib/exportStandard/currencies/`
2. Follow naming: `filenameStructureAndContent.{type}.{format}.ts`
3. Export from: `src/lib/exportStandard/currencies/index.ts`
4. Update documentation in file header

### 6.4. Documentation Updates

When adding new features:

1. Update user guide: `docs/findings/20251228.keyForge.comprehensive.user.guide.md`
2. Add to appropriate section (Advanced Features, Export Options, etc.)
3. Include examples and security considerations
4. Add to FAQ if users might have questions
5. Update troubleshooting section if needed
