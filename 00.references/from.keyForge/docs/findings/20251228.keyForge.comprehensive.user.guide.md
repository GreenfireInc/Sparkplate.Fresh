## 3. Understanding Seed Phrases

### What is a Seed Phrase?

A **seed phrase** (also called a mnemonic phrase or recovery phrase) is a list of 12 or 24 words that represents your cryptocurrency wallet. Think of it as the master key to all your crypto assets.

**Example 12-word seed phrase:**
```
abandon about action adapt adjust admit adult advance advice aerobic afraid again
```

### Why Seed Phrases Matter

- ğŸ”‘ **Master Key**: One seed phrase controls ALL your crypto wallets
- ğŸ”„ **Recovery**: Restore your wallets on any device
- ğŸ›¡ï¸ **Backup**: Physical backup (write it down!)
- ğŸŒ **Universal**: Works with any BIP39-compatible wallet

### 12 Words vs 24 Words

| Feature | 12 Words | 24 Words |
|---------|----------|----------|
| **Security** | 128-bit entropy | 256-bit entropy |
| **Strength** | Very Strong | Extremely Strong |
| **Recommended For** | Personal use | High-value assets |
| **Compatible With** | All wallets | All wallets |

**Recommendation:** 12 words is sufficient for most users. Use 24 words if you're securing large amounts or require maximum security.

### The Checksum

The last word in your seed phrase contains a **checksum** - a mathematical verification that ensures the phrase is valid. If even one word is wrong or in the wrong order, the checksum will fail.

### 5.2. Mnemonic Expansion (Advanced Modal)

The **Advanced** feature allows you to expand a 12-word seed phrase to 24 words.

**Why expand?**
- Increase security from 128-bit to 256-bit entropy
- Meet requirements for certain exchanges
- Personal preference for longer phrases

**Expansion Methods:**

1. **Duplication Method**
   - Converts 12-word entropy to 24-word format
   - Maintains same security base
   - Results in valid BIP39 phrase

2. **Deterministic Method**
   - Combines original entropy with derived entropy
   - Uses SHA-256 hash for additional bits
   - Fully deterministic (same input = same output)

3. **Random Method**
   - Combines original entropy with new random entropy
   - Maximum security (256-bit)
   - Not reproducible from original 12 words

4. **Hybrid Method**
   - Combines deterministic + random entropy
   - Balance of reproducibility and security

**How to use:**

1. Click **"Advanced"** button
2. Enter your 12-word seed phrase
3. Select expansion method
4. Click **"Expand to 24 Words"**
5. View expanded 24-word phrase
6. Verify checksum shows valid
7. Click **"Apply to Main Form"** to use it

âš ï¸ **Important:** Once expanded using Random or Hybrid methods, you cannot reverse the process. The 24-word phrase becomes your new master seed.

## 10. Troubleshooting

### Common Issues & Solutions

#### Issue: "Invalid seed phrase" error

**Possible causes:**
- Typo in one or more words
- Wrong word order
- Non-BIP39 word used
- Incorrect number of words (not 12 or 24)

**Solutions:**
1. Click **"Checksum"** button to validate
2. Review suggested corrections
3. Verify each word against BIP39 wordlist
4. Check for similar-sounding words (e.g., "adapt" vs "adopt")

#### Issue: Generated addresses don't match my wallet

**Possible causes:**
- Different derivation path
- Wrong coin type
- Custom account number
- Using change addresses instead of receiving

**Solutions:**
1. Verify derivation path matches your wallet
2. Check wallet settings for custom derivation paths
3. Try increasing derivation index (some wallets skip index 0)
4. Consult your wallet's documentation

#### Issue: QR code not displaying in exports

**Possible causes:**
- Browser doesn't support Canvas API
- Pop-up blocker preventing download
- Insufficient memory for large exports

**Solutions:**
1. Try different browser (Chrome recommended)
2. Disable browser extensions temporarily
3. Close other tabs to free memory
4. Try exporting in smaller batches

#### Issue: PDF text not selectable

**Solution:**
- This feature was added in version 12.27.2025
- Re-download the latest version of keyForge
- Older PDFs won't have selectable text

#### Issue: Private Key to Mnemonic shows all invalid

**Possible causes:**
- Private key is not 64 hex characters
- Private key includes invalid characters
- Format is incorrect

**Solutions:**
1. Verify private key length (64 characters)
2. Remove any prefixes (`0x`)
3. Ensure only hex characters (0-9, a-f)
4. Try loading from file instead of manual entry

#### Issue: "Generate Addresses" button does nothing

**Possible causes:**
- No seed phrase entered
- Invalid seed phrase
- Browser JavaScript disabled

**Solutions:**
1. Verify seed phrase is entered
2. Check browser console for errors (F12)

4. Try refreshing the page
5. Clear browser cache

## 11. FAQ

### General Questions

**Q: Is keyForge safe to use?**
A: Yes! keyForge runs entirely in your browser with no network requests. All cryptographic operations happen locally. Your seed phrases never leave your device.

**Q: Can I use keyForge offline?**
A: Yes! Save the webpage locally and run it without internet. This is actually recommended for maximum security.

**Q: Is keyForge compatible with hardware wallets?**
A: Yes! keyForge uses standard BIP39/BIP44, making it compatible with Ledger, Trezor, and other hardware wallets. You can import your seed phrase into any compatible device.

**Q: Can I generate wallets for other cryptocurrencies?**
A: Currently, keyForge supports 8 major cryptocurrencies. Support for additional currencies may be added in future versions.

**Q: Do you store my seed phrase?**
A: No. keyForge has no database, no servers, and no storage. Everything happens locally in your browser and is cleared when you close the page.

### Security Questions

**Q: How secure is keyForge compared to hardware wallets?**
A: Hardware wallets are more secure because private keys never touch your computer. keyForge is suitable for generating new wallets, but for large amounts, consider transferring to a hardware wallet.

**Q: What happens if someone gets my seed phrase?**
A: They have complete control of all your crypto. This is why you must protect your seed phrase like cash. Never share it, never photograph it, never store it digitally.

**Q: Can keyForge generate the same address as someone else?**
A: The probability is astronomically low (less than winning the lottery 10 times in a row). BIP39 provides 2^128 possible seed phrases (for 12 words), making collisions virtually impossible.

**Q: Should I generate a new wallet after exporting?**
A: No need. The export itself doesn't compromise security. However, if you've exposed your seed phrase (screenshot, digital storage, etc.), yes, generate a new wallet and transfer funds.

### Technical Questions

**Q: What is the derivation index for?**
A: It allows you to generate multiple addresses from one seed phrase. Index 0 is your first address, index 1 is your second, etc. This is useful for organizing funds or maintaining privacy.

**Q: Why do SOL and XTZ have different derivation paths?**
A: Solana and Tezos use a slightly different derivation path standard. The index position is different, but the principle is the same. keyForge handles this automatically.

**Q: Can I use a seed phrase from another wallet in keyForge?**
A: Yes! As long as it's a valid BIP39 seed phrase, you can import it into keyForge to view your addresses or export backups.

**Q: What's the difference between public key and address?**
A: A public key is the raw cryptographic key. An address is a user-friendly format derived from the public key. You share addresses, not public keys.

**Q: What is GPG key fingerprint?**
A: keyForge generates deterministic GPG (OpenPGP) keys from your wallet keys. The fingerprint is a unique identifier for that GPG keypair. It's useful for encryption and signing operations beyond cryptocurrency.

### Export Questions

**Q: Which export format should I use?**
A: For backups: PDF (print and store physically). For technical use: JSON. For spreadsheets: CSV. For simple text: TXT.

**Q: Can I select text in PDF exports?**
A: Yes! As of version 12.27.2025, all PDF exports include invisible text overlay, allowing you to select and copy addresses, keys, and mnemonics.

**Q: Why does the PDF filename include mnemonic words?**
A: The filename format (e.g., `keyForge.local.20251228.seed.abandonAbout.pdf`) helps you identify which seed phrase the export belongs to without opening the file. Only the first and last words are included (in camelCase).

**Q: Should I encrypt my exports?**
A: YES! If you're storing exports digitally (not recommended), always encrypt them with a strong password. Use tools like 7-Zip, VeraCrypt, or similar.

### Advanced Questions

**Q: How does Mnemonic Expansion work?**
A: It converts a 12-word seed (128-bit entropy) to a 24-word seed (256-bit entropy) using various methods. The original 128 bits are preserved, and 128 bits are added. Different methods add bits differently (deterministic, random, or hybrid).

**Q: What is PIN-based derivation?**
A: Instead of remembering "index 682451", you remember PIN "savings2023". keyForge converts the PIN to an index deterministically. Same PIN always = same index = same addresses.

**Q: How does Private Key to Mnemonic recovery work?**
A: keyForge uses multiple methods to derive possible mnemonics from a private key. It shows you all valid options (those with correct BIP39 checksums). If you know your GPG fingerprint, you can identify the correct one.

**Q: Can I use the same seed phrase for all cryptocurrencies?**
A: Yes! That's the whole point of BIP44. One seed phrase generates addresses for all supported currencies using different derivation paths. This is standard practice.

### Troubleshooting Questions

**Q: My generated BTC address doesn't match my wallet. Why?**
A: Check your wallet's derivation path. keyForge uses standard BIP44 (`m/44'/0'/0'/0/X`), but some wallets use BIP84 (SegWit) or BIP49 (SegWit compatible). The seed phrase is correct, but the derivation method differs.

**Q: Can I recover a wallet if I lost the seed phrase but have the private key?**
A: You can use the **Private Key to Mnemonic** feature to generate possible seed phrases from your private key. However, this doesn't recover your original seed phrase - it creates new ones that happen to contain that private key.

**Q: The checksum validator says my phrase is invalid, but my wallet accepts it. Why?**
A: Your wallet might be using a non-standard implementation. BIP39 has a strict checksum requirement. If the checksum fails, the phrase is technically invalid per the standard.

### E. Glossary

**BIP:** Bitcoin Improvement Proposal - standards for Bitcoin and related technologies

**BIP32:** Standard for hierarchical deterministic wallets (HD wallets)

**BIP39:** Standard for mnemonic seed phrases (recovery phrases)

**BIP44:** Standard for multi-account hierarchy for deterministic wallets

**Checksum:** Mathematical verification that ensures data integrity

**Derivation Path:** The path used to derive a specific key from a seed

**Deterministic:** Same input always produces the same output

**Entropy:** Randomness measure (128-bit = 12 words, 256-bit = 24 words)

**GPG:** GNU Privacy Guard - OpenPGP implementation for encryption

**HD Wallet:** Hierarchical Deterministic wallet - generates keys from single seed

**Mnemonic:** Memory aid - in this context, a seed phrase

**Private Key:** Secret cryptographic key that controls funds

**Public Key:** Public cryptographic key derived from private key

**Seed Phrase:** 12 or 24 words that represent your wallet

**xprv:** Extended Private Key - includes derivation information

**xpub:** Extended Public Key - can derive public keys without private keys
