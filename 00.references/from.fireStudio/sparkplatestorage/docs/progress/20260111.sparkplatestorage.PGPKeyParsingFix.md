# Progress Report: PGP Private Key Parsing Fix

**Date:** January 11, 2026  
**Component:** PGP/GPG Key Tool (App.vue)  
**Status:** ✅ Completed

---

## Overview

Fixed a critical bug in the PGP private key parsing functionality that prevented users from pasting private keys containing literal `\n` escape sequences. The OpenPGP library's `readPrivateKey()` function requires actual newline characters in armored key format, but the application was not converting escape sequences properly.

---

## Problem Description

### Error Encountered

When pasting a PGP private key block containing literal `\n` escape sequences (commonly encountered when copying keys from certain sources), the application threw the following error:

```
Error: readKey: must pass options object containing `armoredKey` or `binaryKey`
```

### Root Cause

The `cleanArmoredKey()` function in `App.vue` was only performing basic cleanup:
- Trimming whitespace
- Removing surrounding quotes

However, it was **not converting literal `\n` escape sequences** (the two-character sequence: backslash followed by 'n') into actual newline characters that the OpenPGP library expects.

### Example Input

When users pasted keys like this:
```
"-----BEGIN PGP PRIVATE KEY BLOCK-----\n\nxVgEX+5mABYJKwYBBAHaRw8BAQdAK0Ec1JvBDS59hiz8qOhdykaq2dNm0TME\n..."
```

The function left the `\n` sequences as literal characters instead of converting them to actual line breaks.

---

## Solution Implemented

### Code Changes

**File:** `src/App.vue`  
**Function:** `cleanArmoredKey()`

#### Before
```typescript
const cleanArmoredKey = (key: string) => {
  return key.trim().replace(/^"|"$/g, '');
};
```

#### After
```typescript
const cleanArmoredKey = (key: string) => {
  let cleaned = key.trim().replace(/^"|"$/g, '');
  // Convert literal \n escape sequences to actual newlines
  cleaned = cleaned.replace(/\\n/g, '\n');
  return cleaned;
};
```

### Technical Details

1. **Regex Pattern:** `/\\n/g`
   - Matches literal backslash-n sequences (`\n` as two characters)
   - Global flag ensures all occurrences are replaced

2. **Replacement:** `'\n'`
   - Converts escape sequences to actual newline characters
   - Required by OpenPGP library for proper armored key parsing

3. **Order of Operations:**
   - First: Trim whitespace and remove quotes
   - Second: Convert escape sequences to newlines
   - This ensures the key format matches OpenPGP expectations
