# Methodology: Cryptocurrency Keypair Decryption — Codebase Locations and Flow

**Date:** April 30, 2026  
**Project:** Sparkplate Storage  
**Scope:** Where **cryptocurrency wallet keypairs** are used to **decrypt** files (SPCK / ECIES envelopes), and how control flows from the UI to crypto primitives.

---

## 1. Summary

Cryptocurrency keypair decryption in this repository is implemented in exactly **two layers**:

| Layer | File | Responsibility |
| ------ | ------ | ---------------- |
| **UI + orchestration** | `src/App.vue` | User picks a file, supplies a private key (or pulls it from the crypto URI field), detects SPCK vs PGP, calls the core decryptor, downloads plaintext. |
| **Cryptography** | `src/lib/cores/cryptographyCore/encryption.cryptoPublicKey.ts` | Parses the binary envelope, validates keys, performs ECDH → HKDF → AES-256-GCM decrypt via Web Crypto. |

There is **no** decryption logic in `src/lib/cores/engine/cryptocurrencyEngine.ts`. That module only parses `%ticker%://%privateKey%` and derives **public** keys and wallet addresses for display; it never reads `.spck` files or calls `decryptWithCryptoPrivateKey`.

## 2. Where Decryption Occurs (Inventory)

### 2.1 Primary implementation — `decryptWithCryptoPrivateKey`

The only function that performs ECIES decryption for a cryptocurrency private key is exported from the cryptography core:

```185:244:src/lib/cores/cryptographyCore/encryption.cryptoPublicKey.ts
export const decryptWithCryptoPrivateKey = async (
  recipientPrivateKey: string | Uint8Array,
  envelope: Uint8Array,
): Promise<Uint8Array> => {
  const subtle = requireSubtleCrypto();

  if (envelope.length < HEADER_LENGTH) {
    throw new Error('Envelope is too short to be a valid SPCK message');
  }

  for (let i = 0; i < SPCK_MAGIC.length; i++) {
    if (envelope[i] !== SPCK_MAGIC[i]) {
      throw new Error('Envelope does not have the expected SPCK magic header');
    }
  }
  const version = envelope[SPCK_MAGIC.length];
  if (version !== SPCK_VERSION) {
    throw new Error(`Unsupported SPCK envelope version: ${version}`);
  }

  const ephemeralPublicKey = envelope.slice(
    SPCK_MAGIC.length + 1,
    SPCK_MAGIC.length + 1 + COMPRESSED_PUBKEY_LENGTH,
  );
  const iv = envelope.slice(
    SPCK_MAGIC.length + 1 + COMPRESSED_PUBKEY_LENGTH,
    HEADER_LENGTH,
  );
  const ciphertext = envelope.slice(HEADER_LENGTH);

  const privateBytes =
    typeof recipientPrivateKey === 'string'
      ? hexToBytes(recipientPrivateKey)
      : recipientPrivateKey;
  if (privateBytes.length !== 32) {
    throw new Error('Private key must be 32 bytes (64 hex chars)');
  }
  if (!secp.utils.isValidSecretKey(privateBytes)) {
    throw new Error('Private key is out of valid secp256k1 range');
  }

  const sharedPoint = secp.getSharedSecret(privateBytes, ephemeralPublicKey, true);
  const sharedX = sharedPoint.slice(1);
  const aesKeyBytes = hkdf(sha256, sharedX, ephemeralPublicKey, HKDF_INFO, 32);

  const aesKey = await subtle.importKey(
    'raw',
    Uint8Array.from(aesKeyBytes),
    { name: 'AES-GCM', length: 256 },
    false,
    ['decrypt'],
  );

  const plaintextBuffer = await subtle.decrypt(
    { name: 'AES-GCM', iv },
    aesKey,
    ciphertext,
  );
  return new Uint8Array(plaintextBuffer);
};
```

**Callers of this function in the repo:** only `src/App.vue` (see §3).

### 2.2 Orchestration and routing — `App.vue`

1. **Imports** — `decryptWithCryptoPrivateKey`, `SPCK_MAGIC`, and `CRYPTO_PUBKEY_FILE_EXTENSION` are imported from `encryption.cryptoPublicKey.ts` (see top of `App.vue`).

2. **`hasSpckMagic`** — Decides whether the selected file is an SPCK envelope (cryptocurrency path) or should fall through to OpenPGP.

3. **`decryptFile`** — Reads the file as bytes; if SPCK magic matches, passes `privateKey` ref (trimmed string) and `fileBytes` to `decryptWithCryptoPrivateKey`; otherwise uses `openpgp`.

4. **`pullPrivateKey`** — Not decryption. It copies the raw 32-byte hex from the cryptocurrency URI field (`cryptoInput`) into the decrypt panel’s `privateKey` textarea by stripping the `TICKER://` prefix. This is how users avoid manual copy-paste of the secret after encrypting with the derived public key.

### 2.3 What does *not* decrypt

| Location | Role |
| --------- | ----- |
| `src/lib/cores/engine/cryptocurrencyEngine.ts` | `deriveCryptocurrency`, `useCryptocurrencyEngine` — parse URI, derive **public** key and address. No envelope I/O. |
| `encryptWithCryptoPublicKey` | **Encrypt** only; symmetric counterpart is `decryptWithCryptoPrivateKey`. |

## 3. End-to-End Flow (Decrypt Button)

The Decrypt File card wires file selection to `decryptFile` and the private key to the shared `privateKey` ref (same ref as the PGP tab uses for armored keys — for SPCK the textarea holds hex).

**Step A — Preconditions**

`decryptFile` requires `fileToDecrypt` and non-empty `privateKey`:

```243:247:src/App.vue
const decryptFile = async () => {
  if (!fileToDecrypt.value || !privateKey.value) {
    decryptionStatus.value = 'Please select a file and provide a private key.';
    return;
  }
```

**Step B — Load bytes and branch**

The entire file is loaded once. Cryptocurrency decryption runs only when the first bytes equal `SPCK` (`SPCK_MAGIC`):

```249:263:src/App.vue
  const file = fileToDecrypt.value;

  try {
    const fileBytes = new Uint8Array(await file.arrayBuffer());

    if (hasSpckMagic(fileBytes)) {
      const trimmedKey = privateKey.value.trim();
      const plaintext = await decryptWithCryptoPrivateKey(trimmedKey, fileBytes);
      const outputName = file.name.replace(
        new RegExp(`\\.${CRYPTO_PUBKEY_FILE_EXTENSION}$`),
        '',
      );
      downloadFile(plaintext, outputName, 'application/octet-stream');
      decryptionStatus.value = 'File decrypted with cryptocurrency private key (ECIES)!';
      return;
    }
```

**Magic check** (same module constant as the encryptor):

```235:241:src/App.vue
const hasSpckMagic = (bytes: Uint8Array): boolean => {
  if (bytes.length < SPCK_MAGIC.length) return false;
  for (let i = 0; i < SPCK_MAGIC.length; i++) {
    if (bytes[i] !== SPCK_MAGIC[i]) return false;
  }
  return true;
};
```

**Step C — Core algorithm** (inside `decryptWithCryptoPrivateKey`)

1. Validate envelope length, magic, version byte.
2. Slice out ephemeral compressed public key (33 bytes), IV (12 bytes), and ciphertext+tag (remainder).
3. Parse `recipientPrivateKey` as 32-byte secp256k1 secret (`hexToBytes` for string).
4. `secp.getSharedSecret(recipientPriv, ephemeralPub)` → take X coordinate as `sharedX`.
5. `hkdf(sha256, sharedX, ephemeralPub, HKDF_INFO, 32)` — must match encryption’s HKDF inputs.
6. `crypto.subtle.importKey` + `decrypt` with AES-GCM; return plaintext `Uint8Array`.

**Step D — Output**

`downloadFile` triggers a browser download. The `.spck` suffix is removed from the filename via `CRYPTO_PUBKEY_FILE_EXTENSION` so the restored file name matches the pre-encryption name pattern.

**Step E — PGP fallback** (not cryptocurrency)

If `hasSpckMagic` is false, execution continues with OpenPGP armored message decode — that path expects ASCII armor, not binary SPCK. Mixing up formats (e.g. feeding `.gpg` as SPCK or vice versa) produces the appropriate errors on the wrong branch.

## 4. Optional UI: Pull Private Key Into Decrypt Panel

Users who encrypt from the cryptocurrency tab often have `cryptoInput` in URI form. **Pull privateKey** does not invoke decryption; it only fills `privateKey` for `decryptFile`:

```56:68:src/App.vue
const pullPrivateKey = () => {
  const raw = cryptoInput.value.trim();
  if (!raw) {
    decryptionStatus.value = 'No private key available to pull yet.';
    return;
  }

  const match = raw.match(/^([A-Za-z0-9]+):\/\/(.+)$/);
  const extracted = match && match[2] ? match[2].trim() : raw;
  privateKey.value = extracted;
  const tickerLabel = match && match[1] ? match[1].toUpperCase() : 'crypto';
  decryptionStatus.value = `Pulled ${tickerLabel} private key into decryption panel.`;
};
```

The regex mirrors the URI shape documented in `cryptocurrencyEngine.ts`; the hex after `://` is exactly what `decryptWithCryptoPrivateKey` expects as a string (optional `0x` is stripped inside `hexToBytes` in the crypto module for keys — the public-key side uses `stripHexPrefix`; private key path uses the same `hexToBytes`).

## 5. Dependency Diagram

```text
[Decrypt File UI]
       │
       ▼
  decryptFile()  ──────────────────────────────────┐
       │                                            │
       ├─ hasSpckMagic(fileBytes)                  │
       │       │                                    │
       │       ├─ true  ──► decryptWithCryptoPrivateKey(priv, envelope)
       │       │                    │
       │       │                    ├─ @noble/secp256k1 (ECDH)
       │       │                    ├─ @noble/hashes HKDF
       │       │                    └─ Web Crypto AES-256-GCM decrypt
       │       │
       │       └─ false ──► openpgp.readPrivateKey + readMessage + decrypt
       │
       └─ downloadFile(plaintext, ...)

[pullPrivateKey] ──► privateKey ref only (no crypto module decrypt call)
[cryptoInput]      useCryptocurrencyEngine ──► derive pub/address only
```
