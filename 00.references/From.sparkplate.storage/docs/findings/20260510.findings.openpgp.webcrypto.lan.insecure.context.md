# 20260510 — Findings: OpenPGP.js "WebCrypto API is not available" over LAN HTTP

**Date:** Sunday, May 10, 2026 (filename date from `date +%Y%m%d` at documentation time)  
**Scope:** Browser error when loading the Vite dev app from another machine via `http://<host-LAN-IP>:<port>`, with stack pointing at `openpgp` / `getWebCrypto`.  
**Outcome:** Dev and preview servers serve **HTTPS** via `@vitejs/plugin-basic-ssl` so `crypto.subtle` is available on remote clients. Use `https://<LAN-IP>:5173` and accept the self-signed certificate warning (or replace with a trusted cert for stricter setups).

---

## Symptom

```
Uncaught Error: The WebCrypto API is not available
    at Object.getWebCrypto (openpgp.js?…)
    …
```

Seen on a **second device** on the same network after `server.host: true` was enabled, while the app works on the dev machine via `http://localhost:5173`.

---

## Root cause

Browsers treat [secure contexts](https://developer.mozilla.org/en-US/docs/Web/Security/Secure_Contexts) as:

- `https://…` origins (with a usable certificate from the browser’s perspective), and  
- **`http://127.0.0.1` / `http://localhost`** (and loopback), as a special case.

A private-network URL such as `http://192.168.x.x:5173` is **not** a secure context. In that situation `globalThis.crypto` may exist, but **`crypto.subtle` is `undefined`**.

[OpenPGP.js](https://github.com/openpgpjs/openpgpjs) v6 resolves Web Crypto through `globalThis.crypto.subtle` and throws if it is missing:

```js
const globalWebCrypto = typeof globalThis !== 'undefined' && globalThis.crypto && globalThis.crypto.subtle;
// ...
throw new Error('The WebCrypto API is not available');
```

So the failure is **not** an OpenPGP bug or a missing polyfill; it is the expected browser policy for HTTP on a non-loopback host.

---

## Solution implemented

1. **Dependency:** `@vitejs/plugin-basic-ssl` (dev dependency).  
2. **Vite:** register `basicSsl()` in `vite.config.ts` so `server.https` and `preview.https` receive a generated self-signed certificate.

After `npm run dev`, open **`https://`** plus the host and port shown under “Network” (for example `https://192.168.0.42:5173`). The first visit will show a certificate warning because the cert is not issued by a public CA; proceed (or install a locally trusted cert, e.g. [mkcert](https://github.com/FiloSottile/mkcert), if you want to avoid warnings).

---

## Operational notes

- **HMR:** If hot reload fails over HTTPS/LAN, see [Vite server.hmr](https://vite.dev/config/server-options.html#server-hmr) (`protocol: 'wss'` and matching client host/port).  
- **Production:** Serve the built app over real HTTPS; the same secure-context rule applies to any code using `crypto.subtle`.  
- **Alternative (not used here):** Tunneling (e.g. Cloudflare Tunnel, ngrok) gives a public HTTPS URL without local TLS setup.
