# Calculator Terra Classic (LUNC) conversion fails — wrong CoinGecko coin id

**Date:** May 29, 2026 (`date +%m%d%Y` → `05292026`)  
**Symptom:** Selecting **Terra Classic (LUNC)** in the calculator “From” card and clicking **Convert** shows:

> Error fetching conversion rates. Please try again.

**Files involved:**

- [`src/components/pageTabs/cryptocurrency/tab.cryptocurrency.Calculator.vue`](../../src/components/pageTabs/cryptocurrency/tab.cryptocurrency.Calculator.vue)
- [`src/lib/cores/bridge/bridge.shared.publicIcons.for.currencies.ts`](../../src/lib/cores/bridge/bridge.shared.publicIcons.for.currencies.ts)
- [`src/lib/cores/currencyCore/currencies/LUNC.TerraClassic.ts`](../../src/lib/cores/currencyCore/currencies/LUNC.TerraClassic.ts)

---

## Root cause

The calculator resolves a selected ticker to a **CoinGecko coin id** before calling:

```http
GET https://api.coingecko.com/api/v3/coins/{id}?market_data=true
```

Flow for LUNC:

1. User picks **LUNC** from the public-icons dropdown (`PUBLIC_ICON_CRYPTO_TICKERS` includes `LUNC`).
2. `cryptoCurrencyFromTicker()` only searched **`COINBASE50`**. LUNC is **not** in that list.
3. Fallback used `{ id: 'lunc', symbol: 'LUNC', name: 'LUNC' }`.
4. CoinGecko has **no** coin with id `lunc` → HTTP **404** → caught in `convertCurrency()` → generic alert.

The correct CoinGecko id is already documented in currency core:

```ts
// LUNC.TerraClassic.ts → identifiers.identifierCoinGecko
'terra-luna-classic'
```

Same class of bug affects other public-icon tickers outside `COINBASE50` when their CoinGecko slug is not the lowercased ticker (e.g. `STX` → `stacks`, not `stx`).

---

## Fix

Added shared resolvers in `bridge.shared.publicIcons.for.currencies.ts`:

| Function | Purpose |
|----------|---------|
| `getCoinGeckoIdForTicker(ticker)` | Resolve slug: `currencyByTicker.identifiers` → `COINBASE50` → `CURRENCY_CORE` → lowercase fallback |
| `cryptoCurrencyFromPublicIconTicker(ticker)` | Build `{ id, symbol, name }` for calculator state |

Updated `tab.cryptocurrency.Calculator.vue` to use `cryptoCurrencyFromPublicIconTicker()` instead of the COINBASE50-only lookup.

For LUNC, `fetchCryptoPrice` now requests `/coins/terra-luna-classic` instead of `/coins/lunc`.

---

## Verification

1. Open **Cryptocurrency → Calculator**.
2. Leave **From** on crypto; select **Terra Classic (LUNC)**.
3. Enter an amount; convert to **USD** (or any fiat).
4. Expect a rate and converted amount — no alert.

Optional devtools check: Network tab should show `…/coins/terra-luna-classic?…` returning **200**.
