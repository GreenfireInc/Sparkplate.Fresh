- [`…coingecko.simple-price-migration.md`](05292026.sparkplate.findings.calculator.coingecko.simple-price-migration.md) — switched to `/simple/price` and added null-safety.

## Root cause — the actual one this time

The previous finding assumed LUNC's CoinGecko slug was `terra-luna-classic`. **That slug does not exist on CoinGecko.** Verified live:

```bash
$ curl "https://api.coingecko.com/api/v3/simple/price?ids=terra-luna-classic&vs_currencies=usd"
{}                       # HTTP 200, empty — id not found
```

CoinGecko's slugs for the two Terra chains are **counter-intuitively inverted** — the *classic* chain kept the original `terra-luna` slug, and the *new* 2.0 chain got the `-2` suffix:

```bash
$ curl ".../simple/price?ids=terra-luna&vs_currencies=usd"
{"terra-luna":{"usd":8.083e-05}}      # ← this is LUNC (Terra Classic)

$ curl ".../simple/price?ids=terra-luna-2&vs_currencies=usd"
{"terra-luna-2":{"usd":0.059811}}     # ← this is LUNA (Terra 2.0)
```

| Token | Chain | **Correct CoinGecko id** |
|-------|-------|--------------------------|
| **LUNC** | Terra Classic (original) | `terra-luna` |
| **LUNA** | Terra 2.0 (new) | `terra-luna-2` |

### Why the calculator hit the empty response

`getCoinGeckoIdForTicker('LUNC')` prefers `currencyByTicker` first (per the first prior fix):

```ts
currencyByTicker['LUNC'].identifiers.identifierCoinGecko  // → "terra-luna-classic"  ❌
```

→ `/simple/price?ids=terra-luna-classic` → `{}` → `data['terra-luna-classic']?.usd` is `undefined`
→ the null-check added in the second fix throws → generic alert.

So the prior two fixes were necessary plumbing, but the underlying **data was wrong**.

---

## The data was inverted in two places

**1. `currencies/LUNC.TerraClassic.ts`** — non-existent slug:

```ts
identifierCoinGecko: "terra-luna-classic",   // ❌ does not exist
```

**2. `indexComposites/currencyCore.ts`** — LUNA and LUNC ids swapped:

```ts
{ id: 'terra-luna',   symbol: 'LUNA', name: 'Terra' },          // ❌ terra-luna IS LUNC
{ id: 'terra-luna-2', symbol: 'LUNC', name: 'Terra Classic' },  // ❌ terra-luna-2 IS LUNA
```

(`LUNA.Terra.ts` was already correct: `identifierCoinGecko: "terra-luna-2"`.)

---

## Fix

**1. `LUNC.TerraClassic.ts`:**

```ts
identifierCoinGecko: "terra-luna",   // ✅ verified live → LUNC price
```

**2. `currencyCore.ts` — un-swap:**

```ts
{ id: 'terra-luna-2', symbol: 'LUNA', name: 'Terra' },          // ✅ LUNA
{ id: 'terra-luna',   symbol: 'LUNC', name: 'Terra Classic' },  // ✅ LUNC
```

After this, every resolver path (`currencyByTicker` preferred, `CURRENCY_CORE` fallback) yields `terra-luna` for LUNC and `terra-luna-2` for LUNA.

---

## Verification

```bash
curl ".../simple/price?ids=terra-luna&vs_currencies=usd"
# → {"terra-luna":{"usd":8.0e-05}}   non-empty, 200
```

In-app:
1. **Cryptocurrency → Calculator**, **From** = crypto.
2. Select **Terra Classic (LUNC)**, amount `1`, **To** = USD → **Convert**.
3. Expect a sub-cent rate (~`0.00008`) and no alert.
4. Also test **LUNA** → expect ~`0.06`, confirming the two are no longer crossed.
