# Finding: Dynamic Tab Canvas Lost on Tab Switch

**Date:** 2026-06-06
**Component:** `modal.Calculator.previewExport.vue` / `tab.Calculator.previewExport.dynamic.vue`
**Severity:** Medium ŌĆö functional regression, silent failure

---

## Symptom

Switching from the **Dynamic** tab to **General** and back causes the Dynamic canvas to appear blank. The preview had rendered correctly on first activation but becomes empty after any tab switch.

---

## Root Cause

### Radix Vue `TabsContent` unmounts on tab switch

Radix Vue's `<TabsContent>` uses conditional rendering (equivalent to `v-if`): when a tab is deselected, its content is **unmounted from the DOM** ŌĆö including any `<canvas>` elements inside it. When the tab is reselected, a brand-new, empty canvas element is created.

### Stale `hasDynamicPreview` guard blocks re-render

`buildDynamicPreview()` is only triggered by the `watch(activeTab)` handler when `!hasDynamicPreview.value`. Because `hasDynamicPreview` was set to `true` during the first render, the guard prevents the function from running again on subsequent switches ŌĆö even though the canvas DOM node it originally drew into no longer exists.

```typescript
// BEFORE ŌĆö guard blocks re-render on every switch after the first
watch(activeTab, async (tab) => {
  if (
    tab === 'dynamic' &&
    props.isOpen &&
    props.snapshot &&
    !hasDynamicPreview.value &&        // ŌåÉ always true after first render
    !isLoadingDynamicPreview.value
  ) {
    await buildDynamicPreview()
  }
})
```

### Sequence of events

| Step | State |
|------|-------|
| 1. Open modal, switch to Dynamic | Canvas mounted, `buildDynamicPreview()` runs, `hasDynamicPreview = true` |
| 2. Switch to General | `TabsContent` unmounts Dynamic content; canvas DOM element destroyed |
| 3. Switch back to Dynamic | New blank canvas mounted; `hasDynamicPreview` still `true` ŌåÆ guard blocks rebuild ŌåÆ blank canvas shown |

---

## Fix

Remove the `!hasDynamicPreview.value` guard so `buildDynamicPreview()` always runs when the Dynamic tab is activated. The function already resets `hasDynamicPreview` to `false` at the start, so it is idempotent.

```typescript
// AFTER ŌĆö always rebuilds when switching to Dynamic tab
watch(activeTab, async (tab) => {
  if (tab === 'dynamic' && props.isOpen && props.snapshot && !isLoadingDynamicPreview.value) {
    await buildDynamicPreview()
  }
})
```

Applied to both `src/components/modals/cryptocurrency/modal.Calculator.previewExport.vue` and `00.backup/modal.Calculator.previewExport.vue`.

---

## Alternative Approaches Considered

| Approach | Trade-off |
|----------|-----------|
| `force-mount` prop on `TabsContent` | Keeps canvas alive but both tab panels are always in the DOM; minor memory cost |
| Reset `hasDynamicPreview` in the General tab watcher | Indirect; same effect but less obvious intent |
| Replace Radix `TabsContent` with `v-show` manually | Breaks Radix accessibility bindings |

The chosen fix is the least invasive and keeps intent clear.
