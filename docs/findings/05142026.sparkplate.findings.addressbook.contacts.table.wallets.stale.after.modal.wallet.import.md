# Contacts table “Wallets” column stays stale after importing wallets from contact details

**Date:** May 14, 2026 (shell: `date -Iseconds` → `2026-05-14T12:07:14-04:00`)  
**Category:** Address book / Vue data flow  
**Status:** Fixed — bubble wallet changes to the view via `contact-updated` + `loadContacts`

## Symptom

After importing wallets through JSON in `subModal.add.Currency.vue` while `modal.details.Contact.vue` is open, the **Contacts** tab (`tab.addressBook.Contact.vue`) kept showing the **old** per-contact wallet count in the **Wallets** column until a full refresh or some unrelated reload.

## Why it happened

1. **`AddressBook.vue`** builds each table row as a `DisplayContact`: base contact fields plus **`wallets`** from `getWalletCountForContact` inside **`loadContacts()`**.
2. **`ContactDetailsModal`** updates its **own** UI after imports via **`handleWalletsImported`** (local `walletCount`, GPG list, wallets tab refresh).
3. The modal **did not** tell the parent view that wallet data for that contact had changed, and **`AddressBook.vue`** only wired **`@close`** on the details modal — **not** a refresh when currencies change.

So the database and the modal were current, but **`contacts`** / **`paginatedContacts`** in the parent were unchanged until something else called **`loadContacts()`**.

## Fix

1. **`modal.details.Contact.vue`** — After a successful single add (**`handleCurrencyAdded`**) and after batch JSON import (**`handleWalletsImported`**), emit **`contact-updated`** (in addition to existing in-modal refreshes).
2. **`AddressBook.vue`** — Handle **`@contact-updated`** with a small helper that:
   - Calls **`loadContacts()`** to recompute **`wallets`** for every row (including the Contacts table), and  
   - If the details modal is still open, replaces **`selectedContactForDetails`** with the matching row from the refreshed list so the modal’s **`contact`** prop does not keep a stale **`wallets`** field.

- `src/components/modals/addressbook/modal.details.Contact.vue` — `handleCurrencyAdded` / `handleWalletsImported`.
- `src/components/modals/addressbook/subModals/subModal.add.Currency.vue` — JSON import entry point.
