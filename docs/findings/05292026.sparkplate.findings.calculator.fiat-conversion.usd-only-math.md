# Calculator non-USD fiat conversion uses USD-denominated rate

**Date:** May 29, 2026 (`date +%m%d%Y` → `05292026`)  
**Symptom:** Converting crypto (or fiat) on the **From** card and selecting a **To** fiat other than USD (e.g. EUR, KES, NGN) on the right card shows amounts as if the target were still **USD** — the label updates to the chosen ISO code but the numeric rate/amount is wrong.

**Files involved:**

- [`src/components/pageTabs/cryptocurrency/tab.cryptocurrency.Calculator.vue`](../../src/components/pageTabs/cryptocurrency/tab.cryptocurrency.Calculator.vue)
- [`src/components/pageTabs/cryptocurrency/aspect.Calculator/aspect.calculator.right.result.vue`](../../src/components/pageTabs/cryptocurrency/aspect.Calculator/aspect.calculator.right.result.vue)

---

## Root cause

Two separate issues; the **conversion math** was the user-visible bug.

### 1. Crypto → fiat assumed USD target (primary bug)

CoinGecko prices are fetched in **USD**:

```ts
/simple/price?ids=bitcoin&vs_currencies=usd  →  current_price in USD
```

The crypto-to-fiat branch treated that USD price as the final rate for **any** target fiat:

```ts
// Before fix — wrong for EUR, KES, etc.
exchangeRate = fromPrice   // USD per coin, shown under target fiat symbol
```

Example: 1 BTC ≈ $100,000 → selecting **EUR** still displayed `100,000` with label `EUR`, instead of ~`92,000` (at ~0.92 EUR/USD).

Fiat-to-crypto had the same USD assumption:

```ts
// Before fix — assumed source fiat was USD
exchangeRate = 1 / toPrice
```

Fiat-to-fiat was already correct (`toRate / fromRate` via exchangerate-api.com).

### 2. `toFiatIso` only synced in child component (secondary)

The searchable fiat dropdown on the right card updates `toFiatIso`, and a watcher in `aspect.calculator.right.result.vue` maps that to `args.to`. The parent `tab.cryptocurrency.Calculator.vue` had no matching watcher (unlike `toCryptoTicker`). In practice the child watcher worked for UI state, but the parent now mirrors the crypto pattern for consistency.

---

## Fix

**1. Apply USD-based fiat cross-rates** (`exchangerate-api.com/v4/latest/USD` — rates = units of currency **per 1 USD**):

| Path | Formula |
|------|---------|
| Crypto → fiat | `exchangeRate = fromPriceUsd × toRate` |
| Fiat → crypto | `exchangeRate = (1 / fromRate) / toPriceUsd` |
| Fiat → fiat | `exchangeRate = toRate / fromRate` *(unchanged)* |
| Crypto → crypto | `exchangeRate = fromPriceUsd / toPriceUsd` *(unchanged)* |

Rates are fetched once per conversion via a shared `getFiatRate()` helper.

**2. Parent watcher** on `toFiatIso` → `args.to` via `fiatByIso` / `toCalculatorFiatOption()`.

---

## Verification

1. **Cryptocurrency → Calculator**; **From** = BTC (or any crypto), amount `1`.
2. **To** = fiat; pick **EUR** (not USD) → **Convert**.
3. Result amount should be ~15–20% lower than USD (typical EUR/USD), not identical to the USD figure.
4. Repeat with **KES** or **NGN** — values should reflect local rates, not raw USD.
5. **Fiat → crypto** with **From** = EUR, **To** = BTC should differ from **From** = USD.
