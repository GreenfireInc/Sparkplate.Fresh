# Address book: JSON wallet import falsely reported as duplicate (`This wallet already exists`)

**Date:** May 14, 2026 (shell: `date -Iseconds` → `2026-05-14T…`)  
**Category:** Address book / contact wallets / event flow  
**Status:** Fixed — import batch must not emit per-wallet `currency-added` after `addWallet`

## Symptom

When importing wallets from JSON in `subModal.add.Currency.vue` (hidden file input → confirm modal → confirm), each imported row triggered:

`alert('This wallet already exists for this contact.')`

from `modal.details.Contact.vue` (`handleCurrencyAdded`), even for addresses that were not previously on the contact.

## Root cause

The JSON import path in `subModal.add.Currency.vue` did two things for each wallet:

1. **`addWallet(...)`** — persisted the row in the data layer immediately.
2. **`emits('currency-added', { contactId, network, address })`** — notified the parent.

The contact details modal handled `currency-added` with `handleCurrencyAdded`, which:

1. Called **`getWalletsForContact`** (now including the row from step 1).
2. Checked **`existingWallets.some((w) => w.coinTicker === network && w.address === address)`**.

Because the wallet had **just** been inserted in the sub-modal, the “existing” list already contained it, so the duplicate check was **always true** for every imported row. No second insert occurred (the handler returned early), but the alert fired once per wallet.

Single-address flow was unaffected: the parent receives `currency-added` **before** any insert and performs the only `addWallet` call.

## Fix (conceptual)

After a batch import with `persistImportedWalletsToContact === true`:

- Keep **only** the in-sub-modal `addWallet` loop.
- Emit **`wallets-imported` once** (no payload required) so the parent runs **`handleWalletsImported`**-style refresh (wallet count, tabs, GPG list) **without** re-running the duplicate guard or calling `addWallet` again.

`modal.details.Contact.vue` already listened for `@wallets-imported="handleWalletsImported"`. The sub-modal now emits that event after the batch instead of N× `currency-added`.

The top-level **`AddressBook.vue`** “add currency” instance of the same sub-modal only listened to `currency-added`; it now also listens to `wallets-imported` and refreshes **`loadContacts()`** so contact wallet counts stay correct when adding currencies from that entry point.
