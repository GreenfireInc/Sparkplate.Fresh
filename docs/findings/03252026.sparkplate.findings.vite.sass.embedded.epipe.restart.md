# Vite Dev Server: `write EPIPE` with `sass-embedded` After Config / Restart Failure

**Date:** March 25, 2026 (`03252026`)  
**Category:** Dev tooling / Build (Vite, Sass)  
**Status:** Operational / environmental (not always a single code bug)

---

## Overview

This finding documents a crash chain that starts with Vite failing to load or restart cleanly, followed by **`Error: write EPIPE`** originating from **`sass-embedded`** (`AsyncCompiler.writeStdin`). The error means the Node process tried to write to a **pipe whose other end was already closed**—typically because a **child process** (here, the embedded Sass compiler) **exited or crashed** while the parent was still sending data.

## What `EPIPE` Means

On Unix-like systems, **`EPIPE`** (“broken pipe”) is returned when a process **writes to a pipe or socket** after the **reader has closed** the other end.

In this stack trace:

1. **`sass-embedded`** runs the Sass compiler as a **separate process** and communicates over **stdin/stdout** (or similar streams).
2. Something causes that **compiler process to exit** (normal exit, crash, OOM, or kill).
3. Node’s `AsyncCompiler` still tries to **`writeStdin`** (send the next chunk of work).
4. The kernel reports **EPIPE** because there is no longer a reader.

So **`EPIPE` is usually a symptom**: the real question is **why the Sass compiler (or Vite’s service) stopped first**.

## How This Relates to the Other Messages

| Message | Likely meaning |
|--------|----------------|
| **Failed to load config from `vite.config.ts`** | Vite could not successfully load or evaluate the config file (syntax/runtime error, missing module, ESM/CJS issue, or transient failure during restart). |
| **The service was stopped: write EPIPE** | An internal Vite/esbuild **service** process ended unexpectedly; a follow-up write hit a broken pipe. |
| **server restart failed** | Hot restart (e.g. after saving `vite.config.ts`) did not complete; state may be inconsistent. |
| **Stack in `sass-embedded`** | While Vite was processing styles or spinning up Sass, the **embedded Sass** child died; the next write threw **EPIPE**. |

These often appear together during **config edits**, **dependency updates**, or **unstable Node versions**, not only from a bug in application source.

---

## Contributing Factors (common)

1. **Node.js version** — **v23.x** is a *Current* (non-LTS) release. Native/tooling stacks (`sass-embedded`, optional binaries) sometimes lag; **LTS (e.g. Node 20 or 22)** reduces odd crashes.
2. **`sass-embedded`** — Uses an embedded compiler process; if that process exits early (bug, resource limit, bad input), **EPIPE** follows.
3. **Vite config reload** — Saving `vite.config.ts` triggers a **full restart**. If the config fails partway or races with Sass, you can see this pattern.
4. **Stale cache** — Corrupt or incompatible **`node_modules/.vite`** cache after upgrades.

## Recommended Responses (solutions)

### 1. Stabilize the dev environment (first)

- **Stop** the dev server completely (Ctrl+C), then start again: `npm run dev`.
- If the first line mentions **config load failure**, open **`vite.config.ts`** and fix any **syntax error**, bad **import**, or **top-level throw** (run `npx vite build` or `node --import tsx` to validate if needed).

### 2. Clear Vite cache

```bash
rm -rf node_modules/.vite
npm run dev
```

### 3. Use an LTS Node for day-to-day development

- Install **Node 20.x or 22.x LTS** (e.g. via `nvm`) and use it for this project.
- Reinstall deps if you switch major Node versions: `rm -rf node_modules && npm install` (with whatever flags the project uses, e.g. `--legacy-peer-deps`).

### 4. If Sass crashes persist: try the pure-JS implementation

The project may list both **`sass`** and **`sass-embedded`**. Vite typically uses `sass` or `sass-embedded` depending on resolution. If **`sass-embedded`** is unstable on your OS/Node combo, prefer **`sass`** (Dart Sass JavaScript API) for development:

- Ensure **`sass`** is installed and lockfile resolves to it for Vite’s pipeline.
- Consult Vite docs for Sass options; avoid duplicate conflicting Sass packages.

*(Do not change `package.json` blindly in production branches without team agreement.)*

### 5. Avoid rapid save loops on `vite.config.ts`

Rapid edits can trigger overlapping restarts. Save once, wait for the server to be ready, then continue.

### 6. Check system resources

Rarely, **memory pressure** kills the Sass child; closing other heavy processes can help.
