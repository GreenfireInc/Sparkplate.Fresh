## Executive summary

Running `vue-tsc` on the repository reports **193** diagnostic lines referencing `currencyCore`, concentrated in **26** distinct `.ts` files (see appendix). The dominant codes are **`TS1005`** (token / `,` / `:` expected), **`TS1160`** (unterminated template literal), and **`TS1127`** (invalid character)—a pattern consistent with **broken template literals and bad embedding of example code inside string literals**, plus at least one case of **non-TypeScript text pasted into a source file**.

These are **parser-level** failures: TypeScript never builds a valid AST for the affected regions, so downstream errors in the same blocks are often **cascading noise** until the first syntax break is repaired.

## Root-cause themes

### 1. Invalid “escaped opening” template literals (`\``…)

Across **`distributionEngines/*/*.smartContract.ts`** (and similar), constants are declared like:

```ts
export const MULTISIG_ESCROW_SCRIPT_EXAMPLE = \`
  …content…
\`;
```

In TypeScript/JavaScript a **leading backslash before `` ` ``** does **not** start a template literal; the lexer treats this as invalid input (`TS1127` / `TS1134`), and the closing `` ` `` leaves the scanner in a bad state (`TS1160` unterminated).

**Correct directions:**

- Use a normal template literal: `` export const X = `...` ``
- Or use a plain double-quoted string with `\n` escapes if no interpolation is needed
- Or move prose (scripts, Solidity, Rust, etc.) to **`.txt` / `.sol` / `.rs` artifacts** (or Markdown under `docs/`) and `import …?raw` or read at build time if the app must bundle them

### 2. Nested “example TypeScript” inside a template literal using `\``

Oracle metadata objects (notably **`oracles/XTZ.Tezos/dia.ts`** and **`kaiko.ts`**) embed long `integration.example` strings that contain lines such as `\`\${API}/path\`` and `console.log(\`...\`)`.

Inside an **outer** template literal, inner backticks must be escaped for **the outer** string only. The pervasive **`\``** pattern mimics Markdown fencing or copy-paste from docs and **breaks parsing** (`TS1160`, then dozens of `TS1005`).

**Solutions:**

- Store examples as **`string` concatenation** of normal single-quoted lines (no nested templates).
- Prefer **Markdown or `.md` snippets** referenced from docs—not executable-looking TS pasted into literals.
- If examples must remain in TS objects, replace inner templates with **concatenation** or **`.replace('{symbol}', …)`** placeholders without backticks inside the outer template.

### 3. Documentation / decorator-like syntax pasted into TS

**`DEXs/DOGE.Dogecoin/stealthex.DOGE.ts`:** Inside what should be `integrationExample: \`…\``, the file contains constructs such as **`@DataSources:`** and array/object fragments that resemble **Angular/Vue-style metadata**, not TypeScript inside a string.

That terminates the conceptual “example block” incorrectly; the remainder of the file is parsed as top-level AST, producing **`TS1160`** at EOF and **`TS1160`/cascade on `stealthex.DOGE.ts(212,1)`**.

**Fix:** Move that block inside a **properly closed** template or normal string; or delete/replace with a short prose `notes` entry and link to external docs.

### 4. Smaller hotspots (still blocking)

Several **oracle** files show similar **comma/colon expected** clusters once a single quote or template boundary is corrupted:

- `oracles/BNB.BinanceCoin/api3.ts`
- `oracles/ETC.EthereumClassic/coingecko.ts`
- `oracles/XLM.Stellar/reflector.ts`
- `oracles/LUNA.Terra/astroport.ts`
- `oracles/ATOM.Cosmos/chainlink.ts`

Treat the **first** reported syntax error per file as the truth; collapse follow-on diagnostics after it is fixed.

## Broader codebase note

Repository-wide **`rg '\\`' *.ts`** under `currencyCore` shows **many** files containing `\`` patterns (often in `oracles/**`, `nftMarketplaces/**`, `stakingPools/**`). Those files **do not** all appear in the current **vue-tsc** error list—they may parse today by accident—or they may fail only when referenced from other compilation units.

**Recommendation:** After fixing the **26 failing** files, add a **`currencyCore`-scoped CI step** (`vue-tsc` or `tsc --noEmit` limited to includes) so new paste-in examples cannot regress the build silently.

## Suggested remediation order

1. **distributionEngines**: Replace all `= \`` / `\`;` opener–closer patterns with valid template literals or externalized raw strings.
2. **DEX StealthEX**: Repair `integrationExample` so no raw `@…` constructs appear outside a closed string literal.
3. **XTZ `dia.ts` / `kaiko.ts`**: Rewrite `integration.example` blobs without nested `\`` escapes; simplify to concatenated strings or move to `.md`.
4. **Remaining oracle singles** (`coingecko`, `reflector`, `astroport`, `chainlink`, `api3`): fix the **primary** unmatched quote/template at the earliest line reported.
5. Re-run **`npx vue-tsc --noEmit --skipLibCheck`** and **`grep '^src/lib/cores/currencyCore'`** until count is zero; then address any **semantic** errors that appear next.
