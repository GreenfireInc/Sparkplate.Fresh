## Root cause

The calculator's `fetchCryptoPrice` function uses the **`/coins/{id}`** endpoint:

```ts
// Before fix
const response = await fetch(
  `https://api.coingecko.com/api/v3/coins/${coinId}?localization=false&tickers=false&market_data=true&...`
)
// ...
return { current_price: data.market_data.current_price.usd }
//                       ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//                       Deep path — throws TypeError if ANY node is null/undefined
```

Three failure modes, any of which shows the generic alert:

| Failure | Cause |
|---------|-------|
| `!response.ok` (429) | CoinGecko free tier: 30 calls/min. A second convert attempt within the window hits the rate limit. |
| `!response.ok` (404) | `/coins/{id}` is stricter; low-cap legacy coins like LUNC may fail on stale cache. |
| `data.market_data.current_price.usd` TypeError | If `market_data` or `current_price` is `null` (partial response, CORS issue, or throttle body), accessing `.usd` throws. No optional chaining. |

Additionally, the `/coins/{id}` response is **~30 KB** of data; the calculator only needs one number.

## Fix

Replace `fetchCryptoPrice` in `tab.cryptocurrency.Calculator.vue` to use `/simple/price`:

```ts
async function fetchCryptoPrice(coinId: string): Promise<{ current_price: number }> {
  const url = `https://api.coingecko.com/api/v3/simple/price?ids=${encodeURIComponent(coinId)}&vs_currencies=usd`
  const response = await fetch(url)

  if (!response.ok) {
    throw new Error(`CoinGecko API returned ${response.status} for "${coinId}"`)
  }

  const data = await response.json()
  const price: number | undefined = (data[coinId] as { usd?: number } | undefined)?.usd

  if (price == null) {
    throw new Error(`No USD price data returned for coin "${coinId}"`)
  }

  return { current_price: price }
}
```

Changes:
- **Endpoint**: `/coins/{id}?...` → `/simple/price?ids={id}&vs_currencies=usd`
- **Response path**: `data.market_data.current_price.usd` → `data[coinId]?.usd` (one level, null-safe)
- **Response size**: ~30 KB → ~50 bytes
- **Rate limit pressure**: Same 30 req/min limit but the lighter endpoint is less likely to be throttled on the free tier.
