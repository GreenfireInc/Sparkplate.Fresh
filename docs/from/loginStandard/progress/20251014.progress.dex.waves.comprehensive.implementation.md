## üîß Technical Highlights

### Blockchain Architecture
- **Consensus:** Leased Proof of Stake (LPoS)
- **Smart Contract Language:** Ride (functional, non-Turing-complete)
- **Block Time:** ~60 seconds
- **Throughput:** ~1,000 TPS
- **Transaction Fee:** 0.003 WAVES per order (fixed)
- **Signatures:** Curve25519 (Ed25519 with X25519 keys)
- **Derivation Path:** N/A (15-word seed phrase, not BIP44)
- **Address Format:** Base58 (starts with 3P for mainnet)

### Unique Waves Features
1. **Native Orderbook DEX:** Built into Waves platform, not separate dApp
2. **Fixed Fees:** 0.003 WAVES per order (no gas wars)
3. **Matcher Nodes:** Centralized matching for speed, non-custodial security
4. **15-Word Seed Phrase:** Waves-specific, not BIP39/BIP44 compatible
5. **Ride Smart Contracts:** Functional language, no loops
6. **Data Oracle System:** Any account can publish oracle data
7. **Leasing:** Stake WAVES by leasing to generating nodes
8. **Custom Tokens:** Issue tokens natively with Issue Transaction

### Integration Methods
1. **Node REST API:** Query blockchain state and broadcast transactions
2. **Matcher API:** Access orderbook and place orders
3. **Data Service API:** Historical data and analytics
4. **Waves Signer:** Sign transactions securely (browser/Node.js)
5. **Waves Transactions:** Create and broadcast transactions
