## Usage Examples

### In Toast Messages
```typescript
// English: "Domain found: vitalik.eth"
// Spanish: "Dominio encontrado: vitalik.eth"
// German: "Domain gefunden: vitalik.eth"
// French: "Domaine trouvé: vitalik.eth"
// Portuguese: "Domínio encontrado: vitalik.eth"
toast.success(`${getTranslation(language, "domainFound")}: ${domainResult.domain}`, {
  description: `${getTranslation(language, "resolvedVia")} ${domainResult.service}`
});
```

### In Tooltips
```typescript
// English: "View on ENS"
// Spanish: "Ver en ENS"
// German: "Auf ENS ansehen"
// French: "Voir sur ENS"
// Portuguese: "Ver no ENS"
title={domainService === 'ENS' 
  ? getTranslation(language, "viewOnEns")
  : getTranslation(language, "viewOnUnstoppableDomains")
}
```

### In Copy Operations
```typescript
// English: "Domain Name"
// Spanish: "Nombre de dominio"
// German: "Domainname"
// French: "Nom de domaine"
// Portuguese: "Nome de domínio"
copyToClipboard(resolvedDomain, getTranslation(language, "domainName"))
```

## Translation Notes

### German (de)
- "Domainname" is written as one word (compound noun)
- "Auf [service] ansehen" is the idiomatic way to say "view on [service]"
- "Aufgelöst über" means "resolved via/through"

### French (fr)
- "Nom de domaine" uses "de" (of) as the preposition
- "Résolu via" is the formal way to express "resolved via"
- Accents are preserved: "Résolution", "Définissez"

### Portuguese (pt)
- Brazilian Portuguese conventions used
- "Nome de domínio" uses "de" as the linking preposition
- "Resolvido via" is the standard technical term

### Spanish (es)
- "Nombre de dominio" uses "de" as the linking word
- "Resuelto vía" correctly uses "vía" with accent
- Latin American Spanish conventions

## Adding New Languages

To add a new language:

1. Create or update `/src/locales/{language-code}.json`
2. Add all 8 domain resolution keys with appropriate translations
3. Test with an Ethereum private key that has an ENS domain
4. Update this reference document with the new translations
