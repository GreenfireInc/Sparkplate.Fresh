## Endpoint Verification Details

### Standard Algorand v2 REST API Endpoints

All implementations correctly use the official Algorand v2 REST API specification:

**Account Information:**
```
GET /v2/accounts/{address}
Response: AlgodAccountInfo | IndexerAccountInfo
```

**Transaction Broadcasting:**
```
POST /v2/transactions
Content-Type: application/x-binary
Body: signed transaction bytes
Response: { txId: string }
```

**Transaction History:**
```
GET /v2/accounts/{address}/transactions?limit={limit}&next={token}
Response: { transactions: [], current-round: number, next-token?: string }
```

**Suggested Transaction Parameters:**
```
GET /v2/transactions/params
Response: { fee, first-valid, last-valid, genesis-id, genesis-hash }
```

**Node/Network Status:**
```
GET /v2/status
Response: { last-round, time-since-last-round, ... }
```
