## Implementation Pattern

Reference implementation: `src/components/modals/confirmations/WalletDeleteConfirmModal.vue`

### Steps to Replace Browser Confirm:

1. Create or reuse a confirmation modal component
2. Add `showDeleteConfirm` ref state
3. Replace direct delete call with `showDeleteConfirm = true`
4. Handle `@confirm` event to perform actual deletion
5. Handle `@close` event to reset state

### Example:

```vue
<template>
  <!-- Delete button triggers modal -->
  <button @click="showDeleteConfirm = true">Delete</button>
  
  <!-- Confirmation modal -->
  <DeleteConfirmModal
    :show="showDeleteConfirm"
    @close="showDeleteConfirm = false"
    @confirm="handleDeleteConfirmed"
  />
</template>

<script setup>
const showDeleteConfirm = ref(false);

const handleDeleteConfirmed = () => {
  showDeleteConfirm.value = false;
  // Perform actual delete
};
</script>
```

---

## Notes

- Native browser `confirm()` dialogs cannot be styled
- They block the JavaScript execution thread
- Custom modals provide better UX and match application design
- Consider creating a reusable `useConfirmDialog` composable for future use
