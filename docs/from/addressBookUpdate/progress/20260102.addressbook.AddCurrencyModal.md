## Problem Summary

## Root Cause Analysis

### Initial Investigation
Initially attempted to fix ContactsTab.vue by:

2. Fixing the event handler to match ContactDetailsModal.vue pattern
3. Correcting import paths for modal components
4. Fixing TypeScript type imports

Through browser testing and codebase analysis, discovered that **ContactsTab.vue is not actually being used in the application**. The application loads `AddressBook.vue` as the main component (see `src/App.vue`).

**Evidence:**
- `src/App.vue` imports and renders `AddressBook.vue`

- ContactsTab.vue appears to be orphaned code

<ActionsDropdown
  :contact="contact"
  @add-currency-request="selectedContactForCurrency = contact; showAddCurrencyModal = true"

  @generate-qrcode-png="onGenerateQrCodePng"

  @export-vcf="onExportVcf"

#### 4. Modal Component (Lines 141-147)
```vue
<AddCurrencyModal
  v-if="selectedContactForCurrency?.id"
  :show="showAddCurrencyModal"
  :contactId="selectedContactForCurrency.id!"
  @close="showAddCurrencyModal = false"
  @currency-added="handleCurrencyAddedFromModal"
/>
```

#### 5. Handler Function (Lines 432-438)
```typescript
async function handleCurrencyAddedFromModal(currencyData: { contactId: number; network: string; address: string }) {
  console.log("Currency added from modal:", currencyData);
  // Add the wallet to the database
  await addWallet(currencyData.contactId, currencyData.network, currencyData.address);
  showAddCurrencyModal.value = false;
  await loadContacts();
}
```

## Technical Details

### Event Flow
1. User clicks on ActionsDropdown in a contact row
2. Selects "Add Currency" from dropdown menu
3. ActionsDropdown emits `add-currency-request` event (no payload)
4. Event handler sets `selectedContactForCurrency` to current contact and `showAddCurrencyModal` to true
5. AddCurrencyModal becomes visible with the contact's ID
6. User fills in currency details (network, wallet address)
7. On submit, modal emits `currency-added` with currency data
8. `handleCurrencyAddedFromModal` function:
   - Calls `addWallet()` to save to database
   - Closes modal
   - Refreshes contacts list to show updated wallet count

### Pattern Consistency
The implementation now matches the pattern used in ContactDetailsModal.vue:
- Inline event handler for `@add-currency-request`
- Conditional rendering with `v-if="contact?.id"`
- Non-null assertion on contactId prop
- Proper cleanup and refresh after currency added
